# 自定义组件优势

1. 体积小
2. 修改样式方便



## 1. 体积小

以`antd`为例，`antd`几乎每个组件都依赖了`react-component`的一个库，如`message`依赖了`rc-notification`、`modal`依赖了`rc-dialog`。所以下载`antd`的时候也会跟着下很多的`rc`包，最后打包的时候`antd`的总体体积会比较大，就算是只用了一个组件，`antd`占用的体积也有一百多k。自定义组件省略了很多三方包，在摇树情况下。最后输出的包体积非常可控



## 2. 样式修改方便

组件库可以提供常用开发场景的绝大多数功能，但是样式确实比较固定的。`CSS` 是前端开发使用较多的场景。不同的项目有不一样的主题风格。

`antd`的部分组件提供了`prefixCls`设置前置类名，方便开发者自定义类型，在可控范围内修改部分样式，但是这种方式修改样式有限，而且仅仅部分组件提供而已。其他无多三方库甚至没有该参数。

使用自定义组件可以极大的方便我们修改样式，我们只需要到`components`下修改组件样式。甚至是重写整个组件的`html、css`。如`message`消息提示组件，几乎所有的前端项目都需要用到，并且功能都是弹出，功能几乎无需改动，我们可以自定义组件中重写整个`message`提示框的样式





# 缺点

1. 当框架改版可能需要改动整个组件库，如从`React18`->`React19`，组件库需要支持 `SSR` 场景下的使用
